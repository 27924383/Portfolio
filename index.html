Why me

Not just another scripter.

Most Roblox scripters can follow a tutorial. Fewer can write a system cleanly from scratch. Fewer still understand both sides of the wire well enough to make the two talk to each other correctly. That gap is where I work.

Dependency count

0 third‑party modules

Every system I've ever shipped was written from the ground up. No ProfileService, no Knit, no DataStore2, no FastCast. No black boxes.

01

Everything is built from scratch

I've never reached for a third-party module because I needed to understand the problem first, then solve it. That means every data system, every replication layer, every hitbox, every ability pipeline — all written by me, all understood by me. No undocumented edge cases hiding inside someone else's code. No library updates breaking your live game.

02

Both sides of the wire, equally

Working only client-side or only server-side is a red flag. The two are deeply coupled — every design decision on one side has consequences on the other. I write full-stack Luau: the server stays authoritative, the client stays responsive, and replication is the deliberate bridge between them, not an afterthought.

03

Code that survives contact with reality

Demos work in empty servers with cooperative players. Real games have 100 players, exploiters sending garbage data, and designers changing the schema six times mid-project. I write for that. Server-authoritative validation, graceful degradation, and predictable behavior under load aren't features I add at the end — they're the starting point.

04

Code you can actually read later

Clever code is a liability. I don't write for impressiveness — I write for the developer who opens the file six months later. Flat structure, clear naming, no unnecessary abstraction. You can hand it to the next person on your team and they won't need me to explain it.

05

Six years of real pattern recognition

After six years you stop guessing and start recognizing. You already know which replication patterns eat bandwidth, which data layouts cause bugs six months later, and which "quick solutions" create problems that take weeks to unwind. That accumulated judgment is what six years actually buys you — and it shows up in every decision, not just the big ones.
